# 原型需求收敛技能

## 何时触发
- 用户上传了 UI 原型/设计稿图片
- 用户执行了 /prototype
- 用户提到"原型图"、"墨刀"、"设计稿"

## 核心原则
截图 ≠ 需求。必须多轮收敛补齐交互、状态、数据、权限等不可见信息。

## 收敛方法论

### 轮次定义
- Round 1 结构层：页面地图、布局、组件清单（必做）
- Round 2 交互层：操作目标、数据来源、分页方式（中等+）
- Round 3 状态层：空态/加载/错误/权限矩阵（中等+）
- Round 4 技术对齐：组件选型、接口可用性（仅复杂）

### 提问规范
- 选择题不用开放题：`A) 跳转 B) 抽屉 C) 弹窗`
- 每轮最多 5-7 个问题
- 每轮输出收敛百分比

### 进入 /plan 门槛
结构100% + 交互>=80% + 状态>=60% + 未决<=3

### 不可见信息推断清单
对每个组件系统性问：交互目标？空状态？加载态？错误态？权限差异？

### 多截图处理
先判断关系（同页不同状态？不同页？），用编号标记，推断流转顺序。

## 与主工作流集成
```
/prototype → (多轮) → /plan → /work → /prototype compare → /compound
```
