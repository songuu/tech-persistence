# 增强记忆管理技能

## 何时触发
- 执行 /learn、/review-learnings、/session-summary、/debug-journal 时
- 用户说"记住这个"、"这是个经验"、"下次注意"
- 解决了一个耗时较长的问题后

## 多层知识管理

### 知识生命周期
```
观察 (observations.jsonl)     ← 自动捕获，最原始
    ↓
自动记忆 (memory/MEMORY.md)   ← v5 启动索引，最多 200 行 / 25KB
    ↓
本能 (instincts/*.md)         ← 从观察中提取，有置信度
    ↓
经验 (rules/*.md)             ← 从本能毕业，成熟知识
    ↓
核心 (AGENTS.md)              ← 最高频、最关键的
```

### 写入决策
```
这条知识是什么层次？
├── 刚发现、未充分验证 → 创建本能 (0.3-0.5)
├── 已验证有效、中等重要 → 写入 rules/ 对应文件
├── 高频、每次都需要 → 写入 AGENTS.md
└── 不确定 → 先创建本能，让系统自动追踪
```

### 本能 vs 经验的区别
- **Memory v5**：轻量、可启动注入，保存未来会用到的命令/调试/流程索引
- **本能**：原子化、有触发条件、有置信度、会衰减
- **经验**：结构化、有上下文/原因/方案、永久保存

### Memory v5 写入规则
- `MEMORY.md` 只放高价值索引，目标 `< 200 行 / 25KB`
- 细节写入 `memory/{topic}.md`，例如 `debugging.md`、`testing.md`、`toolchain.md`
- 写入前先过质量门：有未来价值、无敏感信息、不是单次噪声、与已有条目不重复
- SessionStart 只注入 `MEMORY.md`，topic 文件按需读取

### 毕业条件
本能置信度 >= 0.9 且 30+ 天稳定 → 建议写入 rules/ 成为永久经验

## 去重检查
写入前必须跨层检查：
1. 检查 rules/ 中是否有语义相似的经验
2. 检查 instincts/ 中是否有相似本能
3. 完全重复 → 跳过
4. 部分重复 → 合并，本能置信度提升
5. 矛盾 → 标记旧内容待验证，写入新内容

## 裁剪阈值
- AGENTS.md > 200 行 → 迁移到 rules/
- 单个 rules 文件 > 100 行 → 拆分
- MEMORY.md > 200 行或 >25KB → 裁剪索引，保留 topic 细节
- 本能 > 50 个未聚类 → 建议 /evolve
- 置信度 < 0.15 → 候选删除
- 30 天未更新的 rules → 提醒回顾
